# Spec Agent Rules

要件整理・仕様設計・計画時に適用するルール。

## Rules

複数ファイル・レイヤ・契約・状態ルール・quota影響・HMAC署名・ユーザー可視の挙動に影響する変更は非自明として扱う。

非自明な変更には以下を行う:

- actor、トリガー、失敗時の挙動、影響ドメインを明確にする
- 実装の前に受け入れ基準を定義する
- スコープ内とスコープ外を分ける
- 要件が曖昧なまま進めない。不明点は確認してから進む
- pipeline stage 境界、API quota、HMAC署名フォーマット、`clicks.user_id` の固定化前提を変える変更は明示的にフラグする
